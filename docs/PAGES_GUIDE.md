# Dsphr Home 页面架构与跳转指南
 
本文档详细说明了平台的所有页面、功能描述以及页面间的跳转逻辑。
 
## 1. 核心页面列表
 
目前平台共包含 **14** 个核心路径，细分为约 **40+** 个交互步骤：
 
| 路径 | 页面名称 | 页面/步骤数 | 功能描述 |
| :--- | :--- | :--- | :--- |
| `/` | **欢迎页 (Landing Page)** | 1 | 平台的入口，提供品牌展示与核心功能入口。 |
| `/admin` | **后台管理页 (Admin)** | 1 | 内部管理入口，包含测评目录导航与功能入口。 |
| `/style-eval` | **家居风格测评** | 11 | 10道风格题目 + 1页测评结果。 |
| `/leads` | **项目线索收集** | 2 | 步骤1：项目概况；步骤2：联系信息。 |
| `/home` | **项目中心 (Project Center)** | 1 | 用户管理项目的核心界面，包含 **Sankey 预算看板**。 |
| `/projects` | **项目列表** | 1 | 展示用户关联的所有装修项目。 |
| `/register` | **注册页面** | 1 | 新用户账号创建。 |
| `/login` | **登录页面** | 1 | 已有用户登录。 |
| `/contracts` | **合同签署** | 2 | 步骤1：意向金合同；步骤2：对公转账；**带 `leadId`** 时在步骤2复制账号后同页可选进工作台或深度测评。 |
| `/deep-eval` | **深度需求测评** | 19 | 涵盖空间、功能、习惯等19个详细调研步骤。 |
| `/budget` | **预算看板** | 1 | Sankey 预算流动可视化，按订单/EPC 拆解费用构成，支持在线确认。 |
| `/budget-breakdown` | **初步预算拆解页** | 1 | 基于线索收集生成的初步预算分配建议（饼图展示）。 |
| `/feedback` | **设计反馈** | 1 | 用户对设计方案的反馈与互动。 |
| `/quotation` | **报价单** | 1 | 路由为 **`/quotation`**（无路径参数）；订单上下文由订单详情页 **`navigate` state** 传入。 |
| `/settlement/:id` | **结算单** | 1 | 针对特定订单或阶段的结算详情。 |
| `/order/:id` | **订单详情** | 1 | 展示单个订单的详细状态与内容。 |

## 2. 详细跳转逻辑
 
### 2.1 欢迎页 (Landing Page) - `/` (1页)
*   **“立即开启体验”按钮**：跳转至 `/style-eval`（家居风格测评）。
*   **“进入我的项目”按钮**：已登录 → `/projects`（项目与线索）；未登录 → `/login`，登录成功后进入 `/projects`。登录态本地持久化，刷新欢迎页仍视为已登录。
*   **左下角“设置”图标**：跳转至 `/admin`（后台管理页）。
 
### 2.2 后台管理页 (Admin) - `/admin` (1页)
*   **左上角“返回”按钮**：跳转至 `/`（欢迎页 (Landing Page)）。
*   **“开始风格测评”卡片按钮**：进入风格测评流程（`/style-eval`）。
*   **“去填写项目概况”卡片按钮**：跳转至 `/leads`（线索收集）。
*   **“去注册并填写信息”卡片按钮**：跳转至 `/register`（注册页）。
*   **“开启深度定制之旅”卡片按钮**：跳转至 `/deep-eval`（深度需求测评）。
*   **“进入我的首页”卡片按钮**：跳转至 `/login`（登录页，引导已有账号用户先登录，登录成功后可进入项目中心）。
*   **右上角“测评目录”**：可跳转至导航配置中的任意步骤。
 
### 2.3 家居风格测评 - `/style-eval` (11页)
*   **10道交互题目**：用户选择偏好，点击“下一题”或自动跳转。
*   **1页结果展示**：根据选择生成风格画像。
*   **左上角“返回”按钮**：第一页返回欢迎页，后面每一页就返回上一题。
*   **测评完成后的“去填写项目概况”**：跳转至 `/leads`。
*   **测评完成后的“返回首页”**：返回（欢迎页 (Landing Page)）。
 
### 2.4 线索收集 - `/leads` (2页)
*   **步骤1：项目概况**：填写城市、面积、预算等。
*   **步骤2：您的信息**：填写姓名、电话。
*   **步骤2完成**：跳转至 `/budget-breakdown`（初步预算拆解结果页）。
*   **左上角“返回”按钮**：第一页返回风格测评结果页，后面每一页就返回上一题。
*   **初步预算拆解结果展示**：根据填写生成预算拆解结果（饼图）。点击保存可保存结果，点击注册跳转到注册页。

 
### 2.5 注册/登录 - `/register` / `/login` (各1页)
*   **注册成功**：跳转到合同签署页 `/contracts`。
*   **已有账号去登录**：跳转至 `/login`。（后台管理页「进入我的首页」卡片会优先引导至登录页）
*   **登录成功**：默认跳转 **`/projects`**；若登录入口通过 **`location.state.redirectTo`** 指定路径，则跳转该路径（见 `LoginWrapper`）。
*   **注册页左上角返回**：返回 `/leads?step=2`（线索收集-您的信息）。
*   **登录页返回**：`/`（欢迎页 (Landing Page)）。
 
### 2.7 合同签署 - `/contracts`（2～3 步）
*   **步骤1**：意向金合同阅读与电子签署。
*   **步骤2**：对公转账信息展示，用户可「一键复制全部信息」；复制后出现主操作按钮与「返回主页」按钮；复制按钮约 2 秒后恢复为「一键复制」时，上述按钮仍保留显示。
*   **带 `leadId` 的线索流程**：步骤1 签署后线索状态转为「已转项目」；步骤2 用户复制对公账号信息后，**同页**出现 **进入我的项目**（`enterProjectWorkbenchFromLead` + `/home`）与 **继续深度测评**（`/deep-eval?leadId=…`）。
*   **已签署免重签**：全局已签用户无 `leadId` 时进入合同会直达步骤2；**带 `leadId` 且线索已转项目**时再次打开步骤1 会自动跳到步骤2。`from=home` 且已签署时步骤2 上一步可直接回 `/home`。
*   **来源 A：注册流程**（无 `leadId`、无 `from=home`）：
    *   步骤1 上一步 → `/register`。
    *   步骤2 复制后主按钮「完成支付进入深度测评」→ `/deep-eval`。
    *   「返回主页」→ `/`（欢迎页）。
*   **来源 B：项目中心**（`?from=home`，无 `leadId`）：
    *   步骤1 上一步 → `/home`。
    *   步骤2 主按钮「进入我的项目」→ `/home`；「返回主页」→ `/home`。
    *   不进入步骤3、不进入 `/deep-eval`（除非另行从菜单进入测评）。
*   **来源 C：线索合同**（`/contracts?leadId=…`，从 **`/projects`** 待签线索「去签署」进入）：
    *   步骤1 上一步 → `/projects`。
    *   步骤2 复制对公账号后同页可选 **进入我的项目** / **继续深度测评**；「返回主页」→ `/projects`。
    *   非法 `leadId` 或状态不符时会被重定向回 `/projects`。
*   **步骤2 上一步**：一般回到步骤1；`from=home` 且全局已签署且无 `leadId` 时，步骤2 上一步可直接 → `/home`。
 
### 2.8 深度需求测评 - `/deep-eval` (19页)
*   **19个步骤**：分步调研居住习惯、空间功能等。
*   **全部完成**：跳转至 `/projects`（项目列表）。
*   **第一步 上一步**：返回欢迎页 `/`。
*   **带 `leadId` 进入**：与当前线索绑定（如从合同付款页「继续深度测评」进入）；完成后仍回项目列表等逻辑以页面实现为准。
 
### 2.9 项目列表 - `/projects` (1页)
*   **已签约项目卡片**：点击进入 **`/home`**（载入该线索对应工作台数据）。
*   **待签合同线索**：「去签署」→ **`/contracts?leadId=…`**。
*   **返回按钮**：返回欢迎页 `/`。
 
### 2.10 项目中心 (Project Center) - `/home` (1页)
*   **核心视图**：包含需求、预算、订单、合同等模块的聚合展示。
*   **左上角 Logo**：跳转至 `/`（欢迎页）。
*   **顶部栏“主页”图标**：跳转至 `/`（欢迎页）。
*   **顶部栏“切换项目”图标**：跳转至 `/projects`（项目列表）。
*   **侧边栏导航**（自上而下）：
    *   待定决策（设计反馈等）：已并入 **订单方案** 页顶部，不再单独侧栏入口。
    *   项目协议：原项目合同，查看协议模块；「一键复制账户信息」；「查看合同详情」弹窗；未签署可「去签署」进入 **`/contracts`**（若从线索来可带 `?from=home&leadId=…`，以实际链接为准）。
    *   用户需求：原项目需求书，与采集数据绑定。
    *   预算资金：原预算看板，含 Sankey 与预算确认（亦可从 `/budget` 进入）。
    *   订单方案：原项目订单，列表与 `/order/:id` 详情。
 
### 2.10 项目需求书（项目中心内 tab）
*   **说明**：项目需求书为项目中心（`/home`）侧边栏的 tab 之一，与采集数据完全绑定。
*   **全部为空时**：显示「暂无需求内容」。**从需求书入口进入的用户**（项目概况已由顾问/入项录入）仅需补齐 **家居风格测评** 与 **深度需求测评**：点击「从风格测评开始」→ `/style-eval?from=requirements` → 结果页「开始深度需求测评」→ `/deep-eval?from=requirements&leadId=…`（绑定当前项目线索），**不经过**线索收集、合同签署页。测评目录中亦隐藏线索与合同相关步骤。完成后回到项目中心并自动打开「用户需求」tab。
*   **返回**：返回项目中心并打开 **订单方案**（含顶部待定决策）。
*   **确认并提交**：跳出弹窗，然后转至项目中心主视图。

### 2.11 预算看板 - `/budget` (1页)
*   **功能**：Sankey 预算流动可视化，按订单/EPC 拆解，财务概览与预算方案确认。
*   **返回**：返回 `/home`（项目中心）。

### 2.12 初步预算拆解页 - `/budget-breakdown` (1页)
*   **功能**：展示基于线索收集的初步预算建议（饼图 E/P/C 拆解）；可保存结果。
*   **下一步（注册）**：页内主流程跳转 **`/register`**（与 `BudgetPage` 中 `nextStep` 一致）。
*   **上一步**：返回 **`/leads?step=2`**（您的信息）。
*   **页头左上角返回**：**`/projects`**（项目列表）。
*   右上角 **测评目录** 与其他测评页一致。

### 2.13 项目订单页 - `/order` (1页)
*   **订单卡片**：进入具体的订单详情页`/order/:id`
 
### 2.14 订单详情 - `/order/:id` (1页)
*   **左上角“返回列表”按钮**：返回 `/home`（项目中心-订单 tab）。
*   **设计方案与图纸“查看并反馈”按钮**：进入设计反馈页面 `/feedback`（通过 state 传递订单 ID）。设计反馈页内「返回订单详情」跳转回本页 `/order/:id`。
*   **订单报价单「右箭头」**：进入 **`/quotation`**（通过 **`state`** 传入 `orderNumber`、`orderTitle` 等，非 URL 路径参数）。
*   **订单结算单「右箭头」**：进入 **`/settlement/:id`**（`id` 为订单标识）。

### 2.14.1 设计反馈 - `/feedback` (多步骤)
*   **进入**：由订单详情页「查看并反馈」进入，通过 state 传递订单 ID（`orderNumber`）。
*   **审阅过程中「返回总览」**：返回设计反馈内的订单总览，可再次「继续查看」。
*   **完成审阅后「返回查看」**：返回设计反馈最后一页继续查看。
*   **完成审阅后「返回订单详情」**：跳转至订单详情页 `/order/:id`（与进入时传入的订单 ID 一致）。
*   **总览页「返回订单详情」**：跳转至订单详情页 `/order/:id`。

### 2.15 订单报价单页 - `/quotation` (1页)
*   **路由**：固定为 **`/quotation`**；依赖进入时携带的 **路由 state**（如订单 ID），无则页面可能无法展示完整数据。
*   **左上角 / 弹窗「返回订单详情」**：回到对应的 **`/order/:id`**。

### 2.16 订单结算单页 - `/settlement/:id` (1页)
*   **左上角“返回订单详情”按钮**：返回订单详情页`/order/:id`
*   **弹窗“返回订单详情”按钮**：返回订单详情页`/order/:id`

## 3. 导航组件 (NavigationMenu)
在 `/admin`, `/style-eval`, `/leads`, `/contracts`, `/deep-eval`, **`/budget-breakdown`** 等页面中，右上角均有**测评目录**组件：
*   该组件基于 `src/utils/navigationConfig.ts` 配置。
*   点击目录中的任意项，将根据配置的 `path` 和 `search` 参数进行跳转。
*   当 URL 含 `from=requirements` 且位于 `/style-eval` 或 `/deep-eval` 时，目录仅展示「风格测评 + 深度测评」步骤（不含线索、预算拆解、注册、合同）。
