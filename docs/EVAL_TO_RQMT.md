# 深度测评与项目需求书对应关系

本文档说明 **主链路各环节**（风格测评、线索收集、初步预算拆解、注册、深度测评、合同签署）与 **项目需求书** 各区块之间的对应关系，便于产品、设计与开发理解数据流向与字段映射。**页面顺序**以 `docs/Question Number.md` 为准：风格测评 → 线索 → 初步预算拆解 → 注册 → 深度测评（含风格 q8～q10 移入）→ 合同签署 → 项目列表 → 项目中心。

---

## 1. 总览

| 顺序 | 来源 | 步骤/页数 | 入口路径 | 产出 / 对应需求书 |
|------|------|-----------|----------|--------------------|
| 1 | 家居风格测评 | 8 页（q1～q7 + result） | `/style-eval` | 风格画像、色彩基因、硬装/软装建议 → 需求书「风格人设」「设计定位」 |
| 2 | 线索收集 | 2 步（DE-1、DE-2） | `/leads` | 项目概况、联系信息 → 需求书「项目概览」 |
| 3 | 初步预算拆解 | 1 页 | `/budget-breakdown` | E/P/C 预算建议，不直接写入需求书区块 |
| 4 | 注册 | 1 页 | `/register` | 账号创建，不直接写入需求书 |
| 5 | 深度需求测评 | 22 步（Q2-4～Q2-21，中含风格 q8/q9/q10） | `/deep-eval` | 房型、房屋现状、成员、空间、习惯、设备等 → 需求书各区块 |

## 2. 家居风格测评 → 项目需求书

家居风格测评在 `/style-eval` 路径下，按 `Question Number.md` 为 **文档顺序 1～8**：q1～q7（关于光感/色温/质地/密度/时代/秩序/社交）+ result（风格测评结果页）。q8、q9、q10 已移入深度测评，在 Q2-5 与 Q2-6 之间（见 §4）。

| 产出内容 | 数据示例 | 项目需求书对应区块 |
|----------|----------|--------------------|
| 风格画像 | `style_id`、`style_name`（如 DS01 纯粹极简派、DS02 奶油治愈家、DS14 理性构建者） | **成员画像 → 风格人设**：每位成员的风格偏好标签 |
| 色彩基因 | `colorGene`（浅色系、奶咖色系、大地色系、深色系等） | **设计定位**：色彩与材质基调 |
| 硬装/软装建议 | `suggestions`（硬装、软装、收纳、灯光等） | **结构现状 → 设计定位**：温润克制的高级感、材质与光影优先等 |

**数据流现状**：风格测评结果会保存至 VoyageCard API（`style_id`、`style_name`），LeadsService 支持在提交线索时附带 `design_style_id`、`design_style_name`、`design_style_color`，但当前 `journeySummary.currentFavorite` 未打通。FormData 中暂无风格字段，项目需求书内的「风格人设」「设计定位」仍为 mock 数据，尚未从风格测评结果动态读取。

**相关代码**：`src/pages/StyleEval/`、`src/pages/StyleEval/data/questions.ts`（q1～q10）、`src/services/designVoyage/VoyageCardService.ts`、`src/services/designVoyage/LeadsService.ts`（`buildDesignVoyageLeadPayload` 的 `journeySummary.currentFavorite`）。

## 3. 线索收集 → 项目需求书

线索收集在 `/leads` 路径下，对应 **文档顺序 9～10**：DE-1（项目概况）、DE-2（您的信息），主要产出项目基础信息。

| 步骤 ID | 步骤名称 | 采集内容（FormData / DeepEvalForm） | 项目需求书对应区块 |
|---------|----------|-------------------------------------|--------------------|
| DE-1 / leads-1 | 项目概况 | `projectName`、`projectType`、`projectPosition`、`handoverStatus`、`area`、`budget`、`city` | **项目概览**：项目名称、项目类型、实际面积、预算范围、房屋用途等 |
| DE-2 / leads-2 | 您的信息 | `name`、`salutation`、`phone`、`ageGroup`、`industry` | **项目概览**：业主信息；全局上下文中的用户资料 |

## 5. 项目需求书区块结构

项目需求书按以下结构组织，每个区块标出对应的测评步骤（步骤顺序以 `Question Number.md` 为准，深度测评中含风格 q8/q9/q10）。

```
项目需求书
├── 项目概览
│   ├── 概览信息（城市、类型、面积、预算、入住周期）← DE-1, DE-2, Q2-8
│   ├── 房屋用途 ← Q2-4
│   └── 项目现状（采光、通风、层高、噪音）← Q2-5
├── 结构现状（信息架构、设计定位、交付目标）
│   └── 设计定位（风格、色彩、材质基调）← 家居风格测评（q1～q7+result）、q8 居住定位
├── 项目图纸与视频
│   ├── 户型图 ← Q2-4
│   └── 现场视频/照片 ← Q2-4
├── 成员画像 ← q9 居住成员、q10 家庭爱好、Q2-6, Q2-6-1
│   └── 风格人设 ← 家居风格测评、q10
├── 全屋需求
│   ├── 系统设备（新风、地暖、空调）← Q2-19
│   ├── 智能家居系统 ← Q2-18
│   ├── 全屋设备需求（洗碗机、智能马桶等）← Q2-20
│   ├── 风水要求 ← Q2-17
│   ├── 收纳重点 ← Q2-14
│   └── 个性化定制说明 ← Q2-16, Q2-21
└── 空间需求
    ├── 客厅 ← Q2-13
    ├── 餐厅 ← Q2-12
    ├── 厨房 ← Q2-11
    └── 卫生间 ← Q2-15
```

## 6. 数据流简要说明

主链路顺序（与 `Question Number.md` 一致）：**风格测评 → 线索收集 → 初步预算拆解 → 注册 → 深度测评 → 合同签署 → 项目列表 → 项目中心**。

1. **家居风格测评**：主链路第一步，q1～q7 + result 产出风格画像、色彩基因及硬装/软装建议，保存至 VoyageCard；q8/q9/q10 已移入深度测评（在 Q2-5 与 Q2-6 之间），设计上流入需求书「风格人设」「设计定位」，当前尚未打通。
2. **线索收集**：DE-1、DE-2 产出项目名、类型、面积、预算、城市及业主联系方式等，对应需求书「项目概览」。
3. **初步预算拆解 / 注册 / 合同签署**：不直接写入需求书区块；合同在深度测评之后，签署完成后进入项目列表与项目中心。
4. **深度测评**：按 Q2-4 → Q2-5 → q8/q9/q10 → Q2-6～Q2-21 顺序采集房型、房屋现状、成员、空间与设备等，对应需求书各区块。
5. **项目需求书**：在项目中心侧栏以「项目需求书」形式展示，汇总上述所有采集结果（含风格色彩），供用户查看、编辑与提交。
6. **编辑模式**：需求书支持编辑，可补充户型图、现场视频、智能家居选项、特殊设备、风水、收纳、个性化说明等。
