# Dsphr Home 平台 — 项目需求文档

---

## 1. 文档信息

| 项目 | 说明 |
|------|------|

| 文档用途 | 描述家装设计服务平台的业务需求、功能范围、数据模型与交互流程，供产品、设计与开发统一参考 |

---

## 2. 背景

Dsphr Home 面向家装客户，定位为 **全案项目服务平台**。平台需要系统化采集客户偏好与项目信息，以项目为中心管理订单、合同、预算与交付进度，并支持报价与结算、设计反馈等功能，形成从线索到交付的完整服务链路。

---

## 3. 目标

全案项目客户可以通过产品清晰的了解整个项目及每一个订单的情况，并实现以下功能：

- **需求采集**：通过风格测评、线索收集、深度需求测评，系统化采集客户偏好与项目信息，形成可用的项目需求书与预算依据。  
- **项目与订单管理**：以项目为中心，管理多订单、合同、预算与进度，支持订单状态流转与待定决策展示。
- **报价与结算**：提供订购报价（V1 起）、交付报价、最终结算单的展示、签字与反馈；意向阶段以方案反馈/确认为主，**不提供「意向报价单」**。  
- **设计反馈与维保**：支持设计方案/图纸的查看与反馈，以及订单交付后的维保申请入口。

## 4. 范围

### 4.1 本需求包含

- **功能模块与页面**：欢迎页、家居风格测评、线索收集、初步预算拆解、注册/登录、合同签署、深度需求测评、项目列表、项目中心、用户需求、预算资金、订单方案、订单详情、设计反馈、报价单、结算单；
- **欢迎页**：放于官网首页展示，客户可通过官网点击进行风格测评/进入我的项目。
- **家居风格测评**：7个问题对应7个维度，用户通过测评能拿到测评卡片，需要打通CIL读取问题数据和图片数据。
- **线索收集**：基于CRM的需要采集用户线索信息，并填写进CRM。
- **初步预算拆解**：基于用户输入的信息，调取CRM算法，生成一个初步EPC预算拆解卡片，用户可保存。
- **注册/登录**：用户通过短信验证码的形式注册和登录。
- **合同签署**：用户可查看合同并在线签字签署合同，合同签署后可复制账号信息进行转账。
- **深度需求测评**：通过一系列项目相关的问题，对项目的需求做深入的了解，深度测评的信息最终会生成一个用户需求书。
- **项目列表**：用户名下的项目和线索展示，该页面可供用户增加线索。
- **项目中心**：具体某一个项目的管理中心，包含项目协议、用户需求、预算资金、订单方案等板块，用户可直观浏览项目和订单信息，并进行决策和反馈。
- **用户需求**：用户的需求书，作为项目展开一切的“宪法”，可进行编辑，也可以进行确认，每一次修改都会有历史的修改存档和快照。
- **预算资金**：展示项目预算、订单分配、资金管理的内容，核心是2个预算桑基图的展示，1个是EPC预算拆解桑基图、一个是订单预算拆解桑基图。
- **订单方案**：用于展示订单，客户对于订单的待定决策，桑基图订单拆解的展示，以及订单列表。
- **订单详情**：订单的详情页，包含订单状态、方案、报价单、结算单等相关信息列表，点击可进入子页面。
- **设计反馈**：客户可进入该界面查看订单设计方案，可评论反馈、可通过方案，可查看历史版本。
- **报价单/结算单**：客户可查看报价单/结算单，给予通过确认或者输入评论反馈。报价单/结算单的明细数据从PSO读取。
- **状态跳转**：客户的一些动作会驱动订单状态机的跳转。
- **PSO 对接**：项目订单的数据需要和PSO交互对接，支持读写。

### 4.2 本需求不包含

- 目前仅支持一个项目一个授权决策人可点击查看和反馈，不支持一个项目多个用户查看。
- 订单详情页不做选品的展示。
- 不做签字记忆复用的功能。

## 5. 核心概念

本节为本项目中需要理解的业务与产品概念。

- **线索**：客户填写的项目概况与联系信息，采集后写入 CRM；线索可转为项目，合同签署后线索可转变为项目。
- **项目**：一个全案项目的总称，下挂多笔订单；项目中心即该项目的管理视图，包含项目协议、用户需求、预算资金、订单方案等板块。
- **订单**：项目下的具体交付单元，有状态流转；订单详情页展示订单状态、方案、报价单、结算单等，客户可进入设计反馈、报价/结算子页面。
- **订单状态与阶段**：订单在生命周期内经历意向期、订购期、交付期、验收期、维保期等阶段；不同的阶段对应的状态不一样。具体状态码与流转见 `ORDER_STATE_TRANSITIONS.md`。
- **用户需求书**：由风格测评、线索、深度需求测评汇总而成，作为项目展开的“宪法”；可编辑、确认，每次修改有历史存档与快照，在项目中心「用户需求」板块展示。
- **预算拆解（EPC）**：基于用户输入信息、通过 CRM 算法生成的 E/P/C 初步预算；在「预算资金」中通过 EPC 预算桑基图、订单预算桑基图展示项目预算与订单分配。
- **待定决策**：客户在订单上待完成的决策项（如方案确认、设计反馈、报价/结算签字等），在订单方案板块展示，引导客户进入设计反馈或报价单/结算单操作。
- **报价单与结算单**：包括订购报价（V1 起）、交付报价、最终结算单；按订单阶段展示，客户可查看、通过确认或输入评论反馈；明细数据从 PSO 读取。意向阶段不提供「意向报价单」。
- **合同签署**：客户在线阅读意向金合同并在线签字，签署后可复制对公账号信息完成转账；合同入口包括注册后、从项目中心进入、从线索「去签署」进入，步骤 2 完成后根据来源进入深度测评或项目中心。
- **PSO**：项目与订单数据的系统对接方，项目/订单及报价单、结算单明细等与 PSO 交互，支持读写。
- **CRM**：线索及初步预算算法等数据的对接方，线索信息写入 CRM，初步预算拆解调 CRM 算法。
- **CIL**：风格测评题目与图片等内容的读取来源，需打通以拉取测评题目与图片数据。

---

## 6. 用户角色

- **C 端用户（客户/授权决策人）**：当前为一个项目一个授权决策人，可完成风格测评、线索填写、注册/登录（短信验证码）、合同签署、深度测评；在项目中心查看项目与订单、用户需求、预算资金，处理待定决策；在订单详情查看方案、报价单、结算单并进行设计反馈或维保申请。
- **后台/管理**：通过 admin 等入口进行测评目录导航与流程跳转（视实际需要开放）。

## 7. 系统整体逻辑

### 7.1 主流程概览

- **新用户**：官网欢迎页 → 风格测评（CIL 题目与图片）→ 线索收集（写入 CRM）→ 初步预算拆解（CRM 算法，可保存）→ 注册/登录（短信验证码）→ 深度需求测评（生成用户需求书）→ 合同签署（意向金签字 + 对公转账）→ 项目列表 → 进入项目中心 → 项目协议 / 用户需求 / 预算资金 / 订单方案（含待定决策）→ 订单详情 → 报价单 / 结算单 / 设计反馈。
- **老用户**：欢迎页「进入我的项目」→ 未登录则登录 → 项目列表 → 选择项目 → 项目中心；从项目中心进入合同时。


### 7.2 数据与状态

- 风格测评、线索、深度测评等采集数据汇总为**用户需求书**，在项目中心展示并可编辑与确认，有修改历史与快照。
- **订单及报价单、结算单**与 PSO 对接，订单状态驱动报价单/结算单可见性及客户可执行操作；待定决策由订单状态与客户待办推导，在订单方案板块展示。具体状态与流转见 **§5 核心概念** 及 `ORDER_STATE_TRANSITIONS.md`。
- 用户需求书与深度测评步骤的区块对应关系见 `DEEP_EVAL_TO_REQUIREMENTS_MAPPING.md`。

### 7.3 系统对接

- **CRM**：线索提交与写入；初步预算拆解算法调用。
- **PSO**：项目、订单数据读写；报价单、结算单明细读取；客户动作驱动的订单状态跳转。
- **CIL**：风格测评题目与图片数据读取。
- **auth**：注册、登录（短信验证码）等认证。

### 7.4 路由与页面

#### 7.4.1 页面路由说明

目前平台共包含 **14** 个核心路径，细分为约 **40+** 个交互步骤：
 
| 路径 | 页面名称 | 页面/步骤数 | 功能描述 |
| :--- | :--- | :--- | :--- |
| `/` | **欢迎页 (Landing Page)** | 1 | 平台的入口，提供品牌展示与核心功能入口。 |
| `/admin` | **后台管理页 (Admin)** | 1 | 内部管理入口，包含测评目录导航与功能入口。 |
| `/style-eval` | **家居风格测评** | 11 | 10道风格题目 + 1页测评结果。 |
| `/leads` | **项目线索收集** | 2 | 步骤1：项目概况；步骤2：联系信息。 |
| `/home` | **项目中心 (Project Center)** | 1 | 用户管理项目的核心界面，包含 **Sankey 预算看板**。 |
| `/projects` | **项目列表** | 1 | 展示用户关联的所有装修项目。 |
| `/register` | **注册页面** | 1 | 新用户账号创建。 |
| `/login` | **登录页面** | 1 | 已有用户登录。 |
| `/contracts` | **合同签署** | 2 | 步骤1：意向金合同；步骤2：对公转账；**带 `leadId`** 时在步骤2复制账号后同页可选进工作台或深度测评。 |
| `/deep-eval` | **深度需求测评** | 22 | 涵盖空间、功能、习惯等22个详细调研步骤。 |
| `/budget` | **预算看板** | 1 | Sankey 预算流动可视化，按订单/EPC 拆解费用构成，支持在线确认。 |
| `/budget-breakdown` | **初步预算拆解页** | 1 | 基于线索收集生成的初步预算分配建议（饼图展示）。 |
| `/feedback` | **设计反馈** | 1 | 用户对设计方案的反馈与互动。 |
| `/quotation/:orderId/:ver` | **报价单** | 1 | 路由带订单 id 与报价版本（如 V1/V2/V3）；可选通过 state 传入 `pendingResolveKey`、`quotationTitle` 等。 |
| `/settlement/:id` | **结算单** | 1 | 针对特定订单或阶段的结算详情。 |
| `/order/:id` | **订单详情** | 1 | 展示单个订单的详细状态与内容。 |


#### 7.4.2 页面跳转说明
